## JavaScriptが嫌いなわけじゃない

誤解しないでほしい。JavaScriptは好きだ。プロトタイプを爆速で作れるし、エコシステムは巨大だし、ブラウザでもサーバーでも動く。

でもTypeScriptを使い始めてから、もう戻れなくなった。

## 型は制約じゃなくてドキュメント

`any`を使わない縛りで書くと、関数の引数と返り値が自動的にドキュメントになる。半年後の自分が読んでも「あ、これはこういうデータ構造ね」とすぐ分かる。

```typescript
// これだけで何をする関数か分かる
function calculatePortfolio(
  holdings: StockHolding[],
  prices: Map<string, number>
): PortfolioSummary {
  // ...
}
```

## Zodと組み合わせると最強

フロントとバックエンドでスキーマを共有できる。APIのレスポンスをパースするときに型安全が保証されるのは、精神衛生上とても良い。

## 結論

型を書く手間は、デバッグにかかる時間の10分の1以下。投資対効果は抜群。
